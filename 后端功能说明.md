## 五、自动计算逻辑（重要）

以下字段由后端在新增/修改时自动计算，**前端无需传值，传了也会被覆盖**：

| 字段 | 计算方式 |
|------|----------|
| `yearMonth` | recordDate 年月 → `FYyyMM` 格式（如 2026年7月 → FY2607） |
| `repairHours` | `endTime - startTime`，转为小时 |
| `downtimeHours` | `endTime - repairRequestTime`，转为小时 |
| `lastMachineOnTime` | 在当前表中查找：`machineOffMaterial`（下机物料号）最后一次作为 `machineOnMaterial`（上机物料号）出现的 `recordDate` |
| `isOutOfWarranty` | lastMachineOnTime 为空 → "无"；距今 < 6个月 → "未过保"；≥ 6个月 → "已过保" |
| `agreedRatioQuantity` | `machineCount × unitUsage × ratio` |
| `excessQuantity` | `deliveryQuantity - monthRepair` |
| `excessAmountWithTax` | `excessQuantity × unitPriceWithTax` |

## 六、注意事项

### 6.1 多公司支持
- 所有列表接口都接受可选参数 `companyId`
- **不传 `companyId` = 查看所有公司数据**
- 传了 `companyId` = 只看该公司数据
- 前端流程：加载公司列表 → 渲染下拉框 → 用户选择后传 `companyId` 调用各业务接口

### 6.2 排序字段安全
- `sortField` 参数只能传字母数字下划线，不符合正则 `^[a-zA-Z_][a-zA-Z0-9_]*$` 会被替换为默认值 `id`
- 有效示例：`id`, `recordDate`, `materialCode`, `createdAt`
- 无效示例：`id; DROP TABLE` → 自动降级为 `id`

### 6.3 超比统计的特殊请求格式
- 新增/修改接口请求体需要同时包含主表字段和 `dailies` 数组
- 详见接口文档

### 6.4 操作人追溯
- 操作人从 JWT Token 中提取 `realName`
- `createdBy` = 创建时的操作人
- `updatedBy` = 最后修改时的操作人
- 所有修改操作自动更新 `updatedAt` 时间戳

### 6.5 分页
- 所有列表接口默认 `page=1, pageSize=20`
- 返回 `{total, page, pageSize, list}`
- 前端根据 total 计算总页数：`Math.ceil(total / pageSize)`

### 6.6 CORS
- 后端已配置允许所有来源跨域访问 `/api/**`
- 开发环境前端（如 Vite localhost:5173）可直接调用

## 七、启动方式

```bash
# 1. 确保 MySQL 运行，数据库 metal_system 已创建
# 2. 进入后端目录
cd backend

# 3. 启动
mvn spring-boot:run

# 4. 访问
http://localhost:8080
```

默认管理员账号：联系后端获取（通过 `/api/auth/register` 注册新用户）。
