# 金属厂数据管理系统 — 接口文档

> 后端地址：`http://localhost:8080`
> 所有接口（除登录/注册外）需在请求头携带 Token：
> `Authorization: Bearer <token>`

---

## 统一返回格式

### 成功返回

```json
{
  "code": 200,
  "msg": "success",
  "data": { ... }
}
```

### 分页返回

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "total": 11580,
    "page": 1,
    "pageSize": 20,
    "list": [ ... ]
  }
}
```

### 错误返回

```json
{
  "code": 500,
  "msg": "错误描述",
  "data": null
}
```

## 三、送货记录 `/api/delivery-record`

### 分页列表

```
GET /api/delivery-record?page=1&pageSize=20&companyId=1&keyword=&category=&productAttr=&factory=&startDate=&endDate=&sortField=id&sortOrder=desc
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页条数，默认 20 |
| companyId | Long | 否 | 公司ID，不传返回所有公司数据 |
| keyword | String | 否 | 模糊搜索（物料编码/名称/规格/序列号/品牌） |
| category | String | 否 | 精确筛选类别 |
| productAttr | String | 否 | 精确筛选产品属性（新品/维修品） |
| factory | String | 否 | 精确筛选厂房 |
| startDate | String | 否 | 日期范围起始，格式 yyyy-MM-dd |
| endDate | String | 否 | 日期范围结束，格式 yyyy-MM-dd |
| sortField | String | 否 | 排序字段，默认 id（会做 SQL 注入校验） |
| sortOrder | String | 否 | asc/desc，默认 desc |

**返回数据字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| companyId | Long | 公司ID |
| recordDate | String | 日期 (yyyy-MM-dd) |
| category | String | 类别 |
| materialName | String | 物料名称 |
| specModel | String | 规格型号 |
| materialCode | String | 物料编码 |
| materialSerial | String | 物料序列号 |
| quantity | Integer | 数量 |
| unit | String | 单位 |
| brand | String | 品牌 |
| productAttr | String | 产品属性：新品 / 维修品 |
| factory | String | 厂房 |
| shipmentNo | String | 送货单号 |
| yearMonth | String | 年月，自动生成，格式 FY2607 |
| createdAt | String | 创建时间 |
| updatedAt | String | 更新时间 |
| createdBy | String | 创建人 |
| updatedBy | String | 最后修改人 |

### 详情

```
GET /api/delivery-record/{id}
```

### 新增

```
POST /api/delivery-record
```

> 请求体 JSON 格式同返回字段，只需传业务字段（id/createdAt/updatedAt/createdBy/updatedBy 由后端自动填充）。
> `yearMonth` 由 `recordDate` 自动计算。

### 修改

```
PUT /api/delivery-record/{id}
```

> 传全部字段，后端自动更新 `updatedBy` 和 `yearMonth`。

### 删除

```
DELETE /api/delivery-record/{id}
```

### 批量删除

```
POST /api/delivery-record/batch-delete
```

**请求体：**
```json
{
  "ids": [1, 2, 3]
}
```

### 复制（获取数据用于回填表单）

```
GET /api/delivery-record/copy/{id}
```

> 返回完整数据，前端拿到后去掉 id 即可作为新增请求体。

### Excel 导入

```
POST /api/delivery-record/import
Content-Type: multipart/form-data
```

| 参数 | 类型 | 说明 |
|------|------|------|
| file | File | .xlsx 文件 |

**返回：**
```json
{
  "code": 200,
  "data": {
    "total": 100,
    "success": 95,
    "fail": 5,
    "failDetails": [
      { "row": 3, "reason": "物料编码不能为空" },
      { "row": 7, "reason": "..." }
    ]
  }
}
```

### Excel 导出

```
GET /api/delivery-record/export?companyId=&keyword=&category=&productAttr=&factory=&startDate=&endDate=
```

> 参数与列表查询相同（无分页参数），返回 .xlsx 文件流。浏览器会触发下载。

### 模板下载

```
GET /api/delivery-record/template
```

> 返回含示例数据的 .xlsx 模板，供用户按格式填写后导入。

## 六、送货超比统计 `/api/delivery-stats`

> ⚠️ 该接口接收的非标准 JSON，而是包含嵌套 `dailies` 数组的对象。
> 新增/修改时需同时提交主表数据和每日明细子表数据。

### 分页列表

```
GET /api/delivery-stats?page=1&pageSize=20&companyId=1&keyword=&category=&yearMonth=&sortField=id&sortOrder=desc
```

### 返回字段（主表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | |
| companyId | Long | |
| category | String | 类别 |
| materialCode | String | 物料编码 |
| systemName | String | 系统名称 |
| partName | String | 配件名称 |
| unitUsage | BigDecimal | 单台机用量 |
| ratio | BigDecimal | 比例 |
| unitPriceWithTax | BigDecimal | 含税单价 |
| machineCount | Integer | 机台数 |
| deliveryQuantity | Integer | 送货数量 |
| machineOnQuantity | Integer | 上机数量 |
| monthRepair | Integer | 当月返修 |
| agreedRatioQuantity | BigDecimal | **协议比例数量** = 机台数 × 单台机用量 × 比例（自动计算） |
| excessQuantity | BigDecimal | **超比数量** = 送货数量 - 当月返修（自动计算） |
| excessAmountWithTax | BigDecimal | **超比含税金额** = 超比数量 × 含税单价（自动计算） |

## 附录 B：自动计算字段说明

| 字段 | 所属表 | 计算公式 | 触发时机 |
|------|--------|----------|----------|
| yearMonth | 送货记录/原始记录/上机物料/超比统计 | 从 recordDate/statDate 提取年月，格式化 `FYyyMM` | 新增/修改时 |
| repairHours | 原始记录/上机物料 | `结束时间 - 开始时间`（小时） | 新增/修改时 |
| downtimeHours | 原始记录/上机物料 | `结束时间 - 报修时间`（小时） | 新增/修改时 |
| lastMachineOnTime | 原始记录/上机物料 | 查询同表：当前下机物料号在上机物料列中最后出现的日期 | 新增/修改时 |
| isOutOfWarranty | 原始记录/上机物料 | lastMachineOnTime 距今 \< 6月 → 未过保，≥ 6月 → 已过保，无 → 无 | 新增/修改时 |
| agreedRatioQuantity | 超比统计 | `机台数 × 单台机用量 × 比例` | 新增/修改时 |
| excessQuantity | 超比统计 | `送货数量 - 当月返修` | 新增/修改时 |
| excessAmountWithTax | 超比统计 | `超比数量 × 含税单价` | 新增/修改时 |

> **重要**：这些字段由后端自动计算，前端不需要传值，传了也会被覆盖。

## 附录 C：身份认证

- 登录成功返回 JWT Token，有效期 24 小时
- 所有 API 请求（除 `/api/auth/login` 和 `/api/auth/register`）必须带 Header：
  ```
  Authorization: Bearer <token>
  ```
- 未登录或 Token 过期返回 401
- 所有人权限相同，登录即可操作全部数据
- 操作人姓名自动记录到 `createdBy` / `updatedBy` 字段
