# OCR 图片识别自动填充 — 参考文档

> 基于阿里云 `RecognizeAllText` (通用文字识别高精版) 的实测数据整理。

## 概述

原始记录（金属厂原始记录 / 上机物料）的新增页面支持 OCR 图片识别自动填充。工人拍摄手写工单 → 上传 → 系统调用阿里云 OCR → 自动填表 → 人工校对 → 提交。

## OCR 接口信息

| 项 | 值 |
|---|---|
| 服务商 | 阿里云 OCR |
| 接口 | `RecognizeAllText` (通用文字识别高精版) |
| Endpoint | `ocr-api.cn-hangzhou.aliyuncs.com` |
| 版本 | `2021-07-07` |
| 签名方式 | ACS3-HMAC-SHA256 |
| 图片要求 | jpg/png/bmp, base64 编码后 ≤ 25M, 最短边 ≥ 15px, 最长边 ≤ 8192px |

## 返回数据结构

```json
{
  "Data": {
    "Content": "全文文本...",
    "SubImages": [{
      "Type": "通用文字识别高精版",
      "BlockInfo": {
        "BlockDetails": [
          { "BlockContent": "机台号", "BlockConfidence": 99 },
          { "BlockContent": "ESS",   "BlockConfidence": 93 }
        ]
      }
    }]
  }
}
```

返回的是**按阅读顺序排列的文字块列表**，不是键值对。需要"标签锚点匹配"策略提取字段值。

## 字段提取策略

**核心思路：** 在文字块数组中找到标签（如"机台号"），取其后面的文字块作为值。

```text
Block[i]   = "机台号"  ← 标签，置信度 99%
Block[i+1] = "ESS"     ← 值，置信度 93%
→ 提取为 machineNo = "ESS"
```

**但需注意：**
- 标签可能有多个（如"确认人"出现多次），需要按位置区分
- 手写日期/人名识别率偏低，需要人工校对
- 表格区域（配件列表）提取较复杂

## 字段映射实测（CNC机台维修服务记录单）

### 能准确填充（印刷体标签 + 印刷体/数字值）

| 原始记录表字段 | OCR 识别的标签 | OCR 提取值 | 置信度 | 备注 |
|---|---|---|---|---|
| 厂房 (factory) | "厂房(车间)" | `A` | 99% | 印刷体 |
| 机号 (machineNo) | "机台号" | `ESS` | 93% | 印刷体 |
| 报修时间 (repairRequestTime) | "报障时间" | `2026年6月25日20时00分` | 96% | 印刷体，需转格式 |
| 物料编码 (materialCode) | "配件编码" | `2212673-0461` | 85% | 印刷体，在表格行中 |

### 能提取但需人工校对（手写体）

| 字段 | OCR 提取值 | 问题 | 用户需要 |
|---|---|---|---|
| 维修人 | `何` | 手写名可能识别不全（丢字） | 补全姓名 |
| 确认人 | `欧` | 置信度 72%，可能丢字 | 补全姓名 |
| 开始时间(接单时间) | 手写日期识别乱码 | 手写数字识别率低 | 手工修改 |
| 结束时间(维修结束时间) | 手写日期识别乱码 | 同上 | 手工修改 |

### 工单上没有的字段（OCR 拿不到，只能手工填）

- **班次** (shift) — 工单不标白班/夜班
- **机型** (machineModel) — 工单不一定有
- **上机物料号/下机物料号** — 工单上没有
- **序列号** (serialNumber) — 有一个"设备出厂编号"但不对应

### 总体可用率

- 一张标准 CNC 维修工单约有 **13 个必填字段**
- OCR 能自动填 **4-5 个**（印刷体部分）
- **5-6 个**能提取但需人工改
- **3-4 个**工单上没有，只能手工填

## 前端交互要求

1. **新增页面** → "OCR识别填充"按钮
2. 点击 → 上传图片 → 调用后端 → 字段自动填入表单
3. **OCR 填过的字段高亮标记**（黄色边框或角标），提醒用户校对
4. **低置信度字段特别标记**（红色边框），必须人工确认
5. 用户校对修改后正常提交

## 注意事项

- **只适用原始记录和上机物料**，送货记录不需要 OCR
- OCR 返回的是"建议值"，不是"最终值" — 用户必须人工审核后才能提交
- 手写体识别率受书写质量影响很大，字迹潦草的工单提取效果会打折扣
- 图片分辨率建议 300dpi 以上，拍照时避免反光、倾斜、遮挡
