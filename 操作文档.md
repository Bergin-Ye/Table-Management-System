## 1. 系统概述

金属厂数据管理系统是一套轻量化Web数据管理平台，用于替代传统Excel维护模式，实现数据结构化存储、可视化管理。

### 系统架构

```
浏览器 (Vue 3 + Element Plus)
    ↕ REST API (JWT认证)
后端服务 (Spring Boot 3.3)
    ↕ MyBatis
MySQL 8.4 数据库
```

### 业务模块一览

| 序号 | 模块名称 | 数据量 | 说明 |
|------|----------|--------|------|
| 1 | 送货记录 | 11,580+ | 物料送货全量信息，核心录入表 |
| 2 | 原始记录 | 65,905+ | 金属厂上机/下机原始台账 |
| 3 | 上机物料 | 763+ | 物料上机维修全流程记录 |
| 4 | 送货超比统计 | 186+ | 月度送货超比数据统计 |
| 5 | 结算机台数 | 162+ | 各机型结算机台数统计 |
| 6 | 机型明细 | 3,085+ | 厂房、机台基础信息台账 |
| 7 | 开机数量 | 474+ | 各机型开机数量统计 |
| 8 | 物料表 | 474+ | 物料基础信息（供下拉匹配） |
| 9 | 156项 🆕 | - | 基础物料156项，为其他模块提供自动回填数据源 |

## 2. 快速入门

### 2.1 访问系统

在浏览器中打开系统地址（由管理员提供），出现登录页面。

### 2.2 注册账号

1. 在登录页点击 **"注册"** 标签
2. 填写：用户名、真实姓名、密码（至少6位）、确认密码
3. 点击 **"注册"** 按钮
4. 注册成功后会切换到登录页

> ⚠️ 新注册的用户默认为**普通用户**角色，需要管理员在"用户管理"中提升为管理员。

### 2.3 登录

1. 输入用户名和密码
2. 点击 **"登录"**

### 2.4 默认管理员账号

| 字段 | 值 |
|------|-----|
| 用户名 | **admin** |
| 密码 | **admin123** |

> ⚠️ 首次登录后请立即修改密码！

### 2.5 全局语言设置 🆕

系统所有日期选择器均已配置为**中文**显示：
- 月份：一月 ~ 十二月
- 星期：星期一 ~ 星期日
- 日期格式：yyyy-MM-dd / yyyy-MM-dd HH:mm

> 无需额外设置，登录后自动生效。

### 2.6 界面布局

```
┌─────────────────────────────────────────────┐
│  Header: 系统名称 │ 公司选择 │ 🔔日志 │ 用户头像 │
├────────┬────────────────────────────────────┤
│ 侧边栏  │                                    │
│        │         内容区域                     │
```

- **顶部栏**: 公司选择下拉框、操作日志快捷入口(管理员)、用户菜单
- **侧边栏**: 各模块导航菜单（可折叠）

顶部下拉框可以选择公司，切换后所有页面自动加载对应公司的数据。不选择则显示所有公司数据。

### 5.3 料号自动匹配156项 🆕

在新增或编辑对话框中，**料号**输入框支持从"156项"表智能自动匹配：

1. 在料号框中输入任意关键字
2. 系统自动从**156项**表中模糊搜索匹配的料号
3. 下拉列表显示 `料号 - 配件名称/系统名称`
4. 选中后，系统**自动回填**：
   - **配件名称** — 从156项表带入

> 💡 相比之前的物料表匹配，156项表提供了更精准的配件名称数据。如未查到匹配数据，配件名称保持空值。

### 5.4 过保质保实时查询 🆕

在新增或编辑对话框中，当输入**下机物料号**后：

1. 系统等待输入停顿 0.5 秒后自动查询
2. 查询本表中该物料号最近一次作为**上机物料号**出现的日期
3. 自动显示：
   - **上次上机时间** — 查询结果或无记录显示 `-`
   - **是否过保** — 距上次上机 < 6个月 → `未过保`；≥ 6个月 → `已过保`；无记录 → `无`

> 💡 这是一个纯查询展示功能，不会修改数据。过保状态在提交数据时由后端最终计算并存入数据库。

### 5.5 自动计算逻辑

| 字段 | 计算方式 |
|------|----------|
| **年月** | 从日期提取 → `FYyyMM`（如2026年7月 → FY2607） |
| **维修工时** | `结束时间 - 开始时间`，转换为小时 |
| **停机工时** | `结束时间 - 报修时间`，转换为小时 |
| **上次上机时间** | 查询当前下机物料号在本表"上机物料号"列中最后出现的日期 |
| **是否过保** | 上次上机距今 < 6个月 → "未过保"；≥ 6个月 → "已过保"；无记录 → "无" |

### 5.6 筛选条件

额外支持按班次、是否过保筛选。

> ⚠️ 自动计算字段由后端自动处理，前端无需填写，填写了也会被覆盖。

### 7.2 智能自动回填 🆕

新增/编辑超比统计时，系统支持从"156项"表自动匹配数据并自动查询统计字段：

#### 料号自动匹配

1. 在**料号**输入框中输入关键字
2. 系统自动从**156项**表模糊搜索匹配的料号
3. 下拉列表显示 `料号 - 配件名称/系统名称`
4. 选中后，系统**自动回填**以下字段（来自156项表）：
   - 类别、系统名称、配件名称、单台机用量、比例、含税单价
5. 同时**自动查询并回填**（来自其他业务表）：
   - **机台数** — 从结算机台数表按料号+月份汇总
   - **送货数量** — 送货记录表中当月同料号的记录数
   - **上机数量** — 原始记录表中当月同料号的记录数
   - **当月返修** — 原始记录表中当月同料号且"未过保"的记录数
   - **每日明细** — 当月每一天的送货数量统计

#### 系统名称自动匹配

1. 在**系统名称**输入框中输入文字
2. 系统自动从**156项**表模糊搜索匹配的系统名称
3. 选中后自动回填对应数据（同上）

> 💡 自动查询的字段（送货数量、上机数量、当月返修、每日明细）均为**只读**，不可手动修改。如需修正数据，请在源表中调整后等待定时刷新或重新打开表单。
>
> ⚠️ 通用规则：如自动查询未匹配到数据，对应字段保持空值，不做回填。

### 7.3 自动计算公式

| 字段 | 计算公式 | 触发时机 |
|------|----------|----------|
| **约定比例数量** | `机台数(用户输入) × 单台机用量 × 比例` | 提交时后端计算 |
| **超比数量合计** | `送货数量 − 当月返修` | 提交时后端计算 |
| **超比含税金额合计** | `(含税单价 × 超比数量) ÷ 1.13` | 提交时后端计算 |

> ⚠️ 注意：超比含税金额已自动除以1.13进行去税处理。

### 7.4 汇总行（全部数据）🆕

表格上方显示**全表汇总数据**，按以下公式计算（单位：万元）：

| 汇总项 | 计算公式 |
|--------|----------|
| 送货金额合计 | Σ(含税单价 × 送货数量) ÷ 1.13 ÷ 10000 |
| 上机金额合计 | Σ(含税单价 × 上机数量) ÷ 1.13 ÷ 10000 |
| 返修金额合计 | Σ(含税单价 × 当月返修) ÷ 1.13 ÷ 10000 |
| 比例内金额合计 | Σ(含税单价 × 约定比例数量) ÷ 1.13 ÷ 10000 |
| 超比金额合计 | Σ(含税单价 × 超比数量) ÷ 1.13 ÷ 10000 |

> 📐 公式说明：先将每行的含税单价乘以对应数量，求和后除以 1.13（去税），再除以 10000（换算为万元）。汇总行读取当前**全部数据**（不受分页限制，受筛选条件影响）。

### 7.5 每日明细 🆕

每条统计记录自动展示当月1日至月末最后一天的送货数量明细（根据所选统计日期动态确定天数）。

- 数据来源：自动从**送货记录**表统计当月每一天的送货次数
- 只读展示，不可手动修改
- 新增/编辑时随料号选中自动填充

## 10. 开机数量

**用途**: 各机型开机数量及占比统计，支持基准线自动计算。

### 10.1 表格字段

| 字段 | 说明 |
|------|------|
| 机型 | |
| 数量 | 该机型的开机台数 |
| 占比(%) | **自动计算**（非基准线记录自动根据基准线计算） |
| 统计月份 | 如 2026-07 |
| 备注 | |
| 创建人 | 自动记录 |

### 10.2 基准线机制 🆕

每月有且只有一条**基准线记录**，代表"当月开机总数"，占比固定为 **100%**。

#### 设为基准线

1. 新增或编辑记录时，勾选 **「设为基准线（当月开机总数）」** 复选框
2. 勾选后占比自动设为 100%（不可手动修改）
3. 提交后，系统自动将同月原有的基准线降级，并重新计算同月所有记录的占比

#### 自动重算规则

- **基准线记录**：占比 = 100%
- **其他记录**：占比 = `(该记录数量 ÷ 基准线数量) × 100`
- 基准线的数量变更时，同月所有非基准线记录**自动重算**

#### 删除保护

- 当月存在其他记录时，基准线**不可单独删除**
- 需先将其他记录设为基准线，或删除该月所有记录
- 若强制删除基准线（同月无其他记录时），系统会提示

#### 示例

```
2026-07 月数据：
  机型A: 500台 [✓ 基准线] → 占比 100%
  机型B: 300台            → 占比 (300/500)×100 = 60%
  机型C: 200台            → 占比 (200/500)×100 = 40%
```

> 💡 如果修改机型A的数量为 600，机型B和C的占比会自动更新为 50% 和 33.33%。

## 18. 定时任务 🆕

### 18.1 超比统计数据自动刷新

系统内置了定时任务，**每小时整点**自动刷新当前月份的超比统计数据。

#### 刷新范围

- 仅刷新**当前月份**的超比统计记录（历史月份数据不变）

#### 刷新内容

| 刷新项 | 数据来源 |
|--------|----------|
| 送货数量 | 重新统计送货记录表中当月同料号记录数 |
| 上机数量 | 重新统计原始记录表中当月同料号记录数 |
| 当月返修 | 重新统计原始记录表中当月同料号且"未过保"记录数 |
| 每日明细 | 重新统计每天送货数量 |
| 约定比例数量 | 重新计算：机台数 × 单台机用量 × 比例 |
| 超比数量合计 | 重新计算：送货数量 − 当月返修 |
| 超比含税金额合计 | 重新计算：(含税单价 × 超比数量) ÷ 1.13 |

#### 设计原理

工厂环境下数据持续录入，小时级更新能保证统计数据的时效性。历史月份数据不再有新增记录，无需重复计算。

> 💡 如果在两次定时刷新之间需要立即看到最新统计结果，可以重新打开超比统计的新增/编辑对话框，选中料号后会实时查询最新数据。

## 19. 常见问题

### Q1: 忘记密码怎么办？
联系管理员，管理员在"用户管理"中可以重置密码。

### Q2: 为什么我不能编辑某些数据？
普通用户只能编辑/删除**自己创建**的数据。查看"创建人"列确认数据归属。如需修改他人数据，请联系管理员。

### Q3: 为什么侧边栏有些菜单看不到？
"公司管理"、"操作日志"、"用户管理"仅对管理员可见。如果你应该有管理员权限，请联系现有管理员提升你的角色。

### Q4: 导入Excel失败怎么办？
检查以下几点：
- 是否使用了正确的模板格式
- 必填字段是否填写完整（如物料编码不能为空）
- 日期格式是否正确（yyyy-MM-dd）
- 查看导入结果中的失败详情

### Q5: 数据量大时页面慢怎么办？
- 使用筛选条件缩小查询范围
- 调整每页显示条数
- 导出Excel后离线分析

### Q6: 如何查看谁修改了数据？
管理员在"操作日志"中可以查看所有操作记录，包括操作人、时间、变更内容。

### Q7: 自动计算的字段为什么不显示？
维修工时、停机工时、是否过保等字段由后端自动计算。只有在新增或修改数据时才会触发计算，且必须提供必要的源字段（如开始时间、结束时间）。

### Q8: 超比统计中输入料号后没有自动回填数据怎么办？ 🆕
检查以下几点：
- "156项"表中是否存在该料号的记录（可在156项页面搜索确认）
- 统计日期是否已选择（自动查询依赖日期确定月份）
- 送货记录/原始记录表中当月是否有对应数据（无数据时对应字段为0或空）

### Q9: 156项表料号重复怎么办？ 🆕
系统强制料号唯一：
- 新增/编辑时重复 → 提示"料号已存在"
- Excel导入时重复 → 该行导入失败，出现在失败明细中
- 请先在156项表中检查是否已有该料号

### Q10: 超比统计的每日明细显示为0或不全？ 🆕
每日明细是从送货记录表自动统计的。如果某天没有送货记录，该天数量就是0。数据每小时整点自动刷新一次，如需立即更新请重新打开表单。

### Q11: 结算机台数中点击"选择开机数量"没有数据显示？ 🆕
请先选择**统计月份**，系统会根据月份查询"开机数量"表中的数据。如果该月份没有记录，弹窗表格为空。
